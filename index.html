Question 1: How does javascript work?

Ans: Javascript is an interpreted programming language that works line by line. Every Search engine has its own developed js engine(such as chrome V8) which works for only javascript. When we run a code js engine first check the code if it has any error. If it has any error the engine throws an error about that. And if it has no error then the engine creates a data structure like a tree which is called AST(Abstract Syntax Tree). Then the JavaScript engine turns AST into machine language/code. Because machines don't understand our language until it turns into their own language. Then the engine runs the code.

Question 2: What is the difference between local storage and session storage?

Ans: Local storage and session storage are almost the same. The only difference between these two is local storage data doesn't expire/clear if the page session ends but the session storage data cleared if the page session ends. When we add something to our cart in an E-commerce site after login it is stored in local storage. And those won't be cleared until we remove them from our cart. That's why when we log in our id after many days in that commerce site it shows our cart as like when we added those things in that cart. And when we check something on the website such as plagiarism checker, google translation it stores the data on session storage and when we end the page it clear those data.
